# OpenDraft Workspace Instructions

This workspace contains the OpenDraft reference implementation in multiple languages (C++, .NET, and Java).

## Project Overview

OpenDraft (ODLang) is a language and reference implementation for code generation and templating. The workspace includes:
- **Core libraries** for object models and types
- **Language libraries** for lexical analysis, parsing, and syntax analysis
- **Reference implementations** in C++, .NET, and Java

## Building and Testing

### C++

Build using CMake:
```bash
cmake -S cpp -B cpp/build
cmake --build cpp/build --config Release
```

**Output locations:**
- Static libraries: `out/cpp/lib/`
- Executables: `out/cpp/bin/`

**Key conventions:**
- C++ standard: C++20
- Namespace: `opendraft::` (core, lang subnamespaces)
- Include paths: public headers in `cpp/inc/pub/`, internal in `cpp/inc/`

### .NET

Build and test:
```bash
cd net
dotnet build
dotnet test
```

**Output locations:**
- Compiled assemblies: `out/net/Debug/` or `out/net/Release/`
- NuGet packages: `out/net/packages/`

**Key conventions:**
- Target frameworks: netstandard2.0, netstandard2.1, net10.0
- Nullable reference types: enabled
- Implicit usings: enabled
- LangVersion: latest
- StyleCop rules: configured in `stylecop.json`
- XML documentation: enabled (GenerateDocumentationFile=true)

**MSTest setup:**
- Test projects reference main projects via ProjectReference
- Tests use MSTest framework with async support
- Parallelize at method level

### Java

Build using CMake:
```bash
cmake -S java -B java/build
cmake --build java/build
```

**Output locations:**
- Compiled jars: `out/java/`

**Key conventions:**
- Java version: 21 (Temurin)
- Package names: `io.github.localtoast9001.opendraft.core`, `io.github.localtoast9001.opendraft.lang`
- Source layout: `src/main/java/io/github/localtoast9001/opendraft/`

## GitHub Actions Workflows

Pull requests trigger:
- **cpp-pr.yml**: Configures CMake, builds C++ projects
- **dotnet-pr.yml**: Restores NuGet, builds, and tests .NET projects
- **java-pr.yml**: Sets up Java, configures CMake, builds Java projects

Entry point: `.github/workflows/pr.yml` calls all three reusable workflows.

## Coding Conventions

### C++
- Use `.cpp` or `.c` file extensions for implementation.
- Use `.h` or `.hpp` for header files.
- Namespace declarations: `namespace opendraft::modulename { ... }`
- Both header guards and `#pragma once`
- Include order: system headers, then local headers

### .NET
- File naming matches class names (e.g., `SourceReference.cs`)
- XML doc comments for public types and members
- Use `using` aliases at top of file
- Nullable reference types annotations required

**Exception Testing (MSTest):**
- **Never** use the `[ExpectedException]` attribute on test methods
- Instead, use Try/Catch blocks to validate exception properties and behavior
- Example pattern:
  ```csharp
  [TestMethod]
  public void Constructor_WithNullPath_ThrowsArgumentOutOfRangeException()
  {
      try
      {
          var reference = new SourceReference(null);
          Assert.Fail("Expected ArgumentOutOfRangeException but none was thrown");
      }
      catch (ArgumentOutOfRangeException ex)
      {
          Assert.AreEqual("path", ex.ParamName);
          Assert.IsTrue(ex.Message.Contains("expected message"));
      }
  }
  ```
- This approach provides better control over exception validation and clearer test intent

### Java
- Follow Google Java Style Guide conventions
- Package structure mirrors `io.github.localtoast9001.opendraft.*`
- Javadoc for public classes and methods

## Build Outputs

Generated artifacts are placed under `out/`:
- **C++**: `out/cpp/lib/` (static libraries), `out/cpp/bin/` (executables)
- **.NET**: `out/net/{Configuration}/{TargetFramework}/`
- **Java**: `out/java/` (jar files)

The `out/` directory is excluded from version control.

## CMake Build Directories

`*/build/` directories (e.g., `cpp/build/`, `java/build/`) are generated by CMake and not committed. These are created automatically when you run CMake configure.

## Adding New Features

1. **C++ (core)**: Add `.cpp` files to `cpp/src/core/` or `cpp/src/lang/`, include public headers in `cpp/inc/pub/`
2. **.NET**: Add new projects or classes to `OpenDraft.Core/`, `OpenDraft.Lang/`, or test projects
3. **Java**: Add `.java` files under appropriate package paths in `java/src/main/java/`
4. **Tests**: Add unit tests to language-specific test directories and projects
5. **GitHub Actions**: Update workflow files in `.github/workflows/` if CI needs change
